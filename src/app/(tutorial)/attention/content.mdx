# Paying Attention

<Lead>
In the last chapter, we saw that predicting the next word requires understanding — but our models could only look at a fixed window of recent words. What if the model could *choose* which earlier words to focus on? That idea, called attention, is the breakthrough that makes modern AI work. And it's built entirely from things you already know.
</Lead>

## Some Words Need Other Words

Read this sentence: "The dog chased the cat because **it** was angry."

What does "it" refer to? The dog or the cat? You probably said the dog — dogs chase things because *they're* angry, not the thing being chased. But notice what you just did: to understand one word ("it"), you had to look back at a *specific* earlier word ("dog") and use common-sense reasoning to pick it over another candidate ("cat").

This is something our fixed-window model from the last chapter can't do. It sees the last few words — maybe "because it was" — and has no way to reach back to "dog" at the start of the sentence. It doesn't get to *choose* which earlier words matter.

But for a human reader, this kind of selective looking-back happens constantly. Let's see just how much of language depends on it.

Every one of these examples requires the same ability: given a word, *choose* which other words in the sentence are relevant to understanding it. Not the nearest words. Not a fixed window. The *right* words, wherever they happen to be.

This is what **attention** does. It gives the model the ability to look at every word that came before and decide which ones matter for the word it's currently processing. Let's build up to how it works, step by step.

## Building Attention from Scratch

So how do we actually build a mechanism that does this? We need each word to be able to "ask a question" and find the other words that answer it. Let's start with the simplest possible example.

Imagine a tiny vocabulary of just four tokens: **cat**, **dog**, **bla** (filler), and **it** (a pronoun). Our goal is to work out what noun **it** is referring to in any given sentence.

Each token gets two vectors:

- A **key** — what this token is *advertizing*
- A **query** — what this token is *looking for*

Our keys and queries are just a single number representing "noun-ness" — how much this token is (or wants) a noun:

<ToyVocabTableWidget />

The interesting one is **it**: its key doesn't advertise anything — but its query says "I'm looking for a noun."

To measure how well a query matches a key, we use the **dot product** from Chapter 4 — multiply corresponding numbers and add them up. A high dot product means a good match. Then we run the scores through a function called **softmax** that turns those scores into positive numbers that add up to 100% — we'll explore how softmax works in a moment.

<ToyAttentionWidget>
Click "it" to see its attention computed step by step — its query [3] matches each noun's key [3] (dot product = 9) but not filler's key [0] (dot product = 0). Try "cat bla dog it" — "it" splits attention between both nouns. Try "bla bla bla it" — with no nouns, attention spreads equally over filler. Then try "cat it dog it" — each "it" independently finds the nouns.
</ToyAttentionWidget>

Notice something crucial about **it** — it doesn't advertise anything — but it is looking for a noun. *What you advertize* isn't the same as *what you're looking for.* This is why we need separate queries and keys — a pronoun needs to find nouns without being mistaken for one itself.

In practice, real attention vectors have dozens or hundreds of dimensions, not just one. But the principle is the same — remember the "bear detector" from the vectors chapter, where a single dot product could check for complex things like "something similar to a bear". Attention queries and keys work the same way, just asking richer questions like "is this a noun that's the subject of a nearby verb?"

## Softmax: Dividing Your Attention

Earlier we said that we use **softmax** to turn raw dot-product scores into attention weights. But what does that actually mean, and why do we need it?

Here's the problem. Each token computes a dot product against every other token and gets back a list of raw scores — but those scores can be any numbers: large, small, negative, zero. What we need are *percentages*: each token has a fixed budget of attention — 100% — and needs to divide that budget among all the candidates. We need a way to turn arbitrary numbers into positive weights that add up to 100%.

Softmax is that function. Think of it as a competition: what matters is **who's loudest relative to everyone else**. If one score is moderately high and the others are near zero, it wins by default. But if another score comes along much higher, it drowns out the moderate one and takes nearly all the weight. The *gaps* between scores matter more than the absolute values.

<SoftmaxExplorerWidget>
Start with "All equal" — everyone gets 25%. Then try "Moderate, rest silent" — A isn't very loud, but it wins by default because nobody else is saying anything. Now click "Moderate meets loud" — B comes in much higher and completely drowns out A. That's the key dynamic: softmax is about *relative* loudness. Then try making all scores large but equal (like 8, 8, 8, 8) — still 25% each! It's the gaps that matter, not the magnitudes.
</SoftmaxExplorerWidget>

## Values: What Did You Find?

So far attention tells us *where to look* — which tokens are relevant. But we also need to know *what information to gather* from those tokens.

This is where the third vector comes in. In addition to a **key** (what I advertise) and a **query** (what I'm looking for), each token also gets a **value** — what it *provides* to any token whose query matches its key. Think of it this way: the key and query decide *who talks to whom*, and the value is *what gets said*.

For our pronoun example, the value encodes which animal the token represents. Cat's value says "I'm a cat," dog's value says "I'm a dog," and filler tokens have nothing to contribute:

<ToyValueTableWidget />

Attention then blends these values using the attention weights as a recipe. If "it" pays 90% attention to "cat" and 10% to everything else, the result is 90% cat — the pronoun has figured out what it refers to.

## Multiple Heads: Looking for Different Things

Understanding a sentence requires many kinds of questions at once: What does this pronoun refer to? What's the subject of this verb? What clause does this word belong to?

Real attention models use **multiple attention heads** — each with its own key, query, and value weights, each learning to look for different kinds of relationships. They all run in parallel on the same input, and their outputs get combined. One head might specialize in pronoun resolution (like the BERT head we saw earlier), while another learns to link verbs to their subjects, and another tracks which clause each word belongs to. No one tells these heads what to specialize in — they discover their roles through training, because different kinds of attention are all useful for predicting language.

<KeyInsight>
Multiple attention heads let the model ask many different questions in parallel. Each head has its own key, query, and value weights, so each learns to attend to a different kind of relationship — pronouns, grammar, structure, semantics. **The model doesn't need to choose one kind of attention; it gets all of them at once.**
</KeyInsight>

## What We've Built

Let's take stock. Attention is:

1. **Query, Key, Value** — Each word gets three representations, learned from embeddings
2. **Dot products** — Measure how relevant each word is to each other word
3. **Softmax** — Turn relevance scores into percentages
4. **Weighted sum** — Blend values according to attention percentages
5. **Multiple heads** — Run several attention patterns in parallel, each specializing in different relationships

Every piece is something we already understood from earlier chapters. Attention is just a clever *wiring* of familiar operations.

But there's a problem we haven't solved. Did you notice? Our attention mechanism has no idea *where* words are — it only knows *what* they are. "The dog chased the cat" and "the cat chased the dog" produce identical attention scores. Worse, in a long document, a word at the other end of the book gets treated the same as a word in the same sentence — the model has no sense of proximity at all.
